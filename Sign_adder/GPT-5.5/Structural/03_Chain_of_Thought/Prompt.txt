3. Chain-of-Thought Prompt = 
Prompt- 
Think step by step and reason through each decision before writing Verilog code. 
STRICT RULE: The top-level adder module must be built ONLY through sub-module 
instantiation 
and wire connections. No assign, no always, no arithmetic operators inside the top-level 
module. 
All combinational logic must live inside primitive sub-modules only. 
 
STEP 1 — UNDERSTAND STRUCTURAL MODELING: 
Think: What is structural modeling in Verilog? It is building a module ONLY by instantiating 
other modules and connecting their ports via wires. The top-level module has NO logic of its 
own. 
Why is it useful? It maps directly to physical gate-level netlists used in FPGA synthesis. 
 
STEP 2 — DESIGN THE PRIMITIVE: half_adder 
Think: What is the logic of a half adder? 
  sum  = a XOR b 
  cout = a AND b 
Write a half_adder module using Verilog gate primitives (xor, and). 
 
STEP 3 — DESIGN THE FULL ADDER: full_adder 
Think: How is a full adder built from two half adders? 
  First half adder: s1 = a XOR b, c1 = a AND b 
  Second half adder: sum = s1 XOR cin, c2 = s1 AND cin 
  cout = c1 OR c2 
Write full_adder by instantiating two half_adder modules and one or gate. No assign inside. 
 
STEP 4 — DESIGN THE 16-BIT STRUCTURAL ADDER: adder16_struct 
Think: How do 16 full adders chain together? 
  - Declare wire [16:0] c for the carry chain 
  - c[0] = 1'b0 (no initial carry) 
  - fa0: A[0], B[0], c[0] → sum[0], c[1] 
  - fa1: A[1], B[1], c[1] → sum[1], c[2] 
  - ... continue through fa15 ... 
  - fa15: A[15], B[15], c[15] → sum[15], c[16] 
  - Final output sum[16] = c[16] (overflow/carry bit) 
Write the module with 16 explicit full_adder instantiations. No assign except c[0]=1'b0. 
 
STEP 5 — DESIGN THE TESTBENCH: 
Think: What are ALL corner cases for 16-bit signed addition? 
  (0,0), (32767,0), (-32768,0), (32767,1), (-32768,-1), 
  (32767,32767), (-32768,-32768), (32767,-32768), (-1,1), etc. 
 
Write testbench with: 
  - for loop: 20 explicit corner case pairs, #10 delay, PASS/FAIL check 
  - while loop: 100 iterations, $signed($random) inputs, mismatch tracking 
  - repeat loop: 50 iterations, $signed($random) inputs, display results 
  - forever loop: clk toggle every 5ns, $finish at 1000ns 
 
STEP 6 — VERIFY: 
Re-check: Does adder16_struct contain ONLY module instantiations and wires? 
Is signed on all ports? Is output 17 bits? Are all 4 loops present? Fix any issues.
